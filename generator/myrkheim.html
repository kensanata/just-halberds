The table format

Lines beginning with optional whitespace and a # are comments.

A line starting with a semicolon like ;some words begins a table called ‚Äúsome words‚Äù. There is no space after the semicolon.

A number and a comma like 1,some wordsbegin a weighted entry for the current table.

A text in square brackets with options separated by a vertical bar like [Muspelheim|J√∂tunheim|Vanaheim] gets replaced with one of the options. Single asterisks like *some words* render the text in italics for emphasis.

Double asterisks like **some words** render the text in bold for strong emphasis.

Lines that don‚Äôt start with a semicolon (beginning a table) or a number and a comma (beginning a weighted entry for the current table) are appended to the current entry. A limited form of dice roll notation is recognised and replaced by such a dice roll, e.g. [2d6] or [1d6+3].

The [x?a:b] construct is tricky. You might think of it as ‚Äúpick a or b‚Äù, but a more accurate description would be: ‚Äúexpand both a and b, and then pick‚Äù. One way to rewrite this would be to set the element first and then use indirection.

 They rule a world of [@element].

1,Red salamanders!

;ice salamander
